# 前端开发规则

## 视觉质量标准
使用 Forge 构建的产品必须看起来专业、有质感。不允许出现通用 Bootstrap/模板式的廉价美学。

### 布局
- 页面布局使用 CSS Grid，组件内部使用 Flexbox
- 一致的间距：使用 4px 网格（4, 8, 12, 16, 24, 32, 48, 64）
- 最大内容宽度：1200-1440px，居中容器
- 响应式断点：sm(640px)、md(768px)、lg(1024px)、xl(1280px)
- 移动端优先：先为小屏幕设计，逐步增强到大屏

### 字体排版
- 最多 2 种字体家族（1 个标题字体 + 1 个正文字体）
- 字体大小：使用 UI 库默认的比例
- 正文行高：1.5，标题行高：1.2
- 最大行宽：65-75 个字符以确保可读性

### 颜色
- 在主题配置中定义语义化调色板
- primary、secondary、accent、destructive、muted、background、foreground
- 从第一天起支持深色模式
- 最低对比度比率：正文 4.5:1，大字 3:1（WCAG AA）
- 组件中永远不使用原始十六进制值 — 始终引用主题 token

### 图标
- 统一使用 **Remixicon**（`remixicon` 包）
- 通过 className 引用：`<i className="ri-home-line" />`
- 禁止混用多个图标库

### 组件
- 基于 **shadcn/ui** 构建
- 每个交互元素需要：
  - 悬停状态
  - 焦点状态（可见的焦点环）
  - 按下/激活状态
  - 禁用状态（适用时）
  - 加载状态（异步操作时）
- 按钮：异步操作时始终显示加载旋转图标
- 表单：行内验证，清晰的错误信息，正确的标签
- 表格：可排序表头，分页，空状态，加载骨架屏
- 列表：空状态，加载骨架屏，错误状态

### 动画与动效
- 过渡：微交互 150ms，布局变化 300ms
- 缓动：进入用 ease-out，退出用 ease-in
- 不做装饰性动画 — 每个动效都应该传达状态变化
- 尊重 `prefers-reduced-motion`

### 无障碍
- 所有图片有 alt 文本
- 所有表单输入有标签（可见或 sr-only）
- 所有交互元素支持键盘导航
- 模态框、抽屉和动态内容有焦点管理
- 语义 HTML 不够时使用 ARIA 属性
- 颜色永远不是唯一的指示器（同时使用图标/文字）

## 状态管理
- 局部状态：`useState` 用于组件级状态
- 共享状态：使用 **Zustand** 管理全局状态
- URL 状态：使用路由参数/查询参数表示可分享的状态
- 永远不要在客户端复制服务器状态 — 缓存它，不要拷贝它

## API 集成
- HTTP 客户端统一使用 **Axios**，配置统一的 baseURL、拦截器、错误处理
- **禁止手动编写 RESTful 接口调用代码** — 所有 API 请求由 `@umijs/openapi` 基于后端 Swagger 文档自动生成
- 接口生成流程：后端启动 → 前端执行 `pnpm openapi` → 生成 `src/services/` 下的接口文件和类型
- 为每个数据获取处理加载、错误和空状态
- Axios 拦截器统一处理 token 注入、401 跳转、错误提示

## 测试
- 工具函数和复杂 hooks 的单元测试
- 交互组件的组件测试
- 关键用户流程的集成测试
- 使用技术栈配置中指定的测试库
