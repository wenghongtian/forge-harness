# 第一性原理思维

## 含义
第一性原理思维意味着将问题分解到最基本的事实，然后从这些事实出发构建方案。不要复制现有方案 — 先理解它们存在的原因，再判断同样的逻辑是否适用于你的情况。

## 如何应用

### 产品设计
在设计任何功能之前，回答以下问题：
1. **用户真正想要完成什么？**（不是"他们想要什么功能"，而是"他们需要什么结果"）
2. **达到该结果的最短路径是什么？**（去掉所有仪式感）
3. **哪些约束是真实的，哪些是假设的？**（挑战每一个"我们必须"的说法）
4. **一个新用户会期望什么？**（新鲜的视角，没有习得性无助）

### 架构设计
在选择任何技术或模式之前，回答：
1. **实际的数据流是什么？**（先画数据流图，再选框架）
2. **真实的扩展需求是什么？**（不是"如果我们爆火了怎么办"，而是"第一年的量级"）
3. **什么变化最频繁？**（把它隔离在接口后面）
4. **最简单的可行方案是什么？**（然后在真实条件下验证它是否真的可行）

### 需要避免的反模式
- "大家都用 X，所以我们也应该用" — 流行度不是理由
- "以后可能会需要" — YAGNI，除非有具体证据
- "一直都是这样做的" — 用数据挑战传统
- "我们加一个抽象层吧" — 只在有 3 个以上具体场景时才抽象
- "微服务因为要扩展" — 先用单体，感受到真实痛点再拆分

### 决策框架
对于每个重要决策，记录：
1. **问题** — 具体要解决什么？
2. **约束** — 什么是固定的，什么是灵活的？
3. **选项** — 至少 3 个，包括"什么都不做"
4. **推理** — 为什么选这个，从第一性原理出发
5. **取舍** — 我们放弃了什么？
