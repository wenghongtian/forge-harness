# UI/UX 设计标准

## 何时使用
当实现前端界面时引用此技能。确保每个页面和组件都达到专业视觉水准。

## 视觉设计检查清单

### 布局
- [ ] 使用 4px 间距网格（4, 8, 12, 16, 24, 32, 48, 64）
- [ ] 内容区域最大宽度 1200-1440px
- [ ] 移动端优先的响应式设计
- [ ] 页面有清晰的视觉层次（标题 > 副标题 > 正文 > 辅助文字）

### 颜色
- [ ] 所有颜色使用语义化 token（primary, secondary, muted, destructive）
- [ ] 没有硬编码的十六进制值
- [ ] 正文与背景对比度 ≥ 4.5:1（WCAG AA）
- [ ] 深色模式下颜色正确反转

### 字体
- [ ] 最多 2 种字体家族
- [ ] 正文行高 1.5，标题行高 1.2
- [ ] 正文最大行宽 65-75 字符

### 组件状态完整性
每个交互组件必须有以下状态：
- **默认** — 正常显示
- **悬停** — 鼠标移入反馈
- **焦点** — 键盘聚焦时的可见环
- **按下** — 点击瞬间反馈
- **禁用** — 不可操作时的灰态
- **加载** — 异步操作进行中

### 页面状态完整性
每个数据驱动的页面必须处理：
- **加载中** — 骨架屏，不是空白页
- **数据正常** — 正常展示
- **空数据** — 有意义的空状态（图标 + 文案 + 操作引导）
- **错误** — 错误提示 + 重试按钮
- **部分失败** — 成功的部分正常显示，失败的部分单独提示

## 交互模式

### 表单
- 行内实时验证（失焦时触发，不是每次输入）
- 错误信息紧跟对应字段下方，红色文字
- 提交按钮在异步操作时显示加载态并禁用
- 成功后用 Toast 提示，不要用 alert

### 列表/表格
- 超过 20 条数据要分页
- 提供排序和筛选（至少支持关键字段）
- 空列表显示引导性空状态
- 加载时显示骨架屏（3-5 行）

### 模态框
- 打开时聚焦到第一个可交互元素
- ESC 键关闭
- 点击遮罩关闭（除非是确认类模态）
- 关闭时焦点回到触发元素

### 通知/反馈
- 成功操作：绿色 Toast，3 秒自动消失
- 失败操作：红色 Toast，需要手动关闭
- 危险操作（删除等）：二次确认弹窗

## 响应式断点

| 断点 | 宽度 | 布局 |
|------|------|------|
| mobile | < 640px | 单列，汉堡菜单 |
| tablet | 640-1023px | 可折叠侧栏 |
| desktop | 1024-1279px | 完整侧栏 |
| wide | ≥ 1280px | 完整侧栏 + 更宽内容区 |

## 无障碍要求（WCAG 2.1 AA）

- 所有图片有 `alt` 文本
- 所有表单有可见或 `sr-only` 的 `label`
- Tab 键可以遍历所有交互元素
- 焦点环清晰可见（不要 `outline: none`）
- 颜色不是唯一的信息载体（配合图标或文字）
- 动画尊重 `prefers-reduced-motion`
